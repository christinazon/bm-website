06 — How we did this

Methodology

Source
British Museum's own API
Data pulled from britishmuseum.org/collection — the same system that powers their public search. Every on-display object's "Production place" field was read individually.
One country per object
No overlaps, no guessing
We used the "Made in:" field specifically — not the broader place tag (which includes findspot) and not keyword search. 1,270 unique place names mapped to modern countries.
Known gap
46% have no origin recorded
The museum itself hasn't filled in the production place for nearly half of on-display objects — mostly prehistoric and ancient material. Reported honestly, not guessed at.
~7% overlap
Date-bucket boundary effect
The API caps results at 10,000 per query. We split retrieval into 11 date buckets. Objects with fuzzy date ranges (e.g. "200 BC–AD 200") may appear in two adjacent buckets. Affects all countries equally.
45,185 vs ~80,000
The museum's own figures differ
The British Museum publicly states ~80,000 objects are on display. Our search returned 45,185. The gap likely reflects incomplete digital records for some galleries.
Findspot
Kept strictly separate
Where objects were physically found is not the same as where they were made. The two datasets are never mixed — findspot is supplementary context, not a production-place substitute.
